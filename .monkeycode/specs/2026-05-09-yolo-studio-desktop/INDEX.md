# 标炬（LabelTorch）文档总览

## 文档清单

1. `prd.md`：产品需求文档
2. `requirements.md`：EARS 风格需求约束
3. `design.md`：技术设计总文档
4. `blueprint.md`：版本蓝图（MVP 到成熟版）
5. `tasklist.md`：工程任务书
6. `file-architecture.md`：文件层级与模块规划
7. `product-roadmap-detailed.md`：详细版本路线图
8. `system-architecture.md`：系统架构分册
9. `data-architecture.md`：数据与模型版本架构
10. `workflow-architecture.md`：业务与训练工作流架构
11. `nonfunctional-requirements.md`：非功能与稳定性标准
12. `boundary-and-assumptions.md`：边界限制与假设
13. `qa-and-release-governance.md`：测试与发布治理
14. `packaging-and-distribution.md`：打包与分发规划
15. `risk-register.md`：风险清单与缓解策略
16. `ai-execution-contract.md`：给实现模型的执行协议
17. `../../../../reference/REFERENCE_LINKS.md`：外部参考项目链接清单

## 使用顺序

1. 先读 `prd.md` 与 `requirements.md`
2. 再读 `system-architecture.md` 与 `data-architecture.md`
3. 然后按 `product-roadmap-detailed.md` + `tasklist.md` 执行
4. 实施过程中遵守 `ai-execution-contract.md` 与 `qa-and-release-governance.md`
5. 参考实现前先阅读 `reference/REFERENCE_LINKS.md` 并按优先级选择仓库
