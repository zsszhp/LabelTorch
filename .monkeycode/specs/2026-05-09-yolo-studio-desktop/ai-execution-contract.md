# 标炬（LabelTorch）AI 实施执行协议

## 目标

确保任意实现模型按统一约束开发，避免偏离规划。

## 输入约束

1. 必须先读取 `INDEX.md`。
2. 必须遵守 `requirements.md` 与 `boundary-and-assumptions.md`。
3. 每次只实现一个里程碑。
4. 开始实现前必须先阅读 `reference/REFERENCE_LINKS.md`，并在任务说明中声明本轮参考了哪些仓库。

## 输出约束

1. 每次提交必须附：
   - 实现范围
   - 改动文件清单
   - 验收结果
   - 风险与待办
2. 不得引入越界功能。

## 里程碑执行顺序

1. M1 基础骨架
2. M2 数据与标注
3. M3 训练与版本
4. M4 辅助标注与增量
5. M5 导出与发布

## 验收规则

1. 不通过验收禁止进入下一里程碑。
2. 出现 P0/P1 缺陷必须先修复。
3. 任何数据破坏风险改动必须提供回滚策略。

## 变更控制

1. 若新增需求超出当前版本边界，必须先更新 PRD 与蓝图文档。
2. 架构层改动必须同步更新 `system-architecture.md` 与 `file-architecture.md`。
