# 标炬（LabelTorch）产品蓝图

## 1. 产品阶段规划

### V0.1 MVP

目标：打通离线训练与增量标注最小闭环。

功能：

1. YOLO txt 导入与校验
2. 类别映射重排
3. bbox 编辑
4. 训练与早停
5. 辅助标注与批量确认
6. 版本管理与增量训练
7. pt/onnx 导出
8. Windows 双包发布

### V0.2

目标：增强训练效率与实验管理。

功能：

1. 断点续训
2. 自动混合精度
3. 实验对比看板
4. COCO/VOC 基础导入

### V0.3

目标：提升大规模训练能力与 OBB 深度支持。

功能：

1. 多卡训练
2. 超参搜索
3. OBB 标注交互增强
4. 数据健康度分析

### V0.4

目标：扩展任务类型。

功能：

1. 分类任务标注与训练
2. 异常检测任务
3. 多任务统一项目空间

### V1.0

目标：成熟稳定发布。

功能：

1. 完整异常恢复机制
2. 插件化模型适配层
3. 长期版本维护策略

## 2. 技术路线图

1. 单机单进程 UI + 多子进程任务执行
2. 先 bbox，再扩展 OBB
3. 先单卡稳定，再扩展性能特性
4. 先 YOLO 生态闭环，再扩展多任务

## 3. 风险与应对

1. GPU 环境差异
   - 应对: CPU/CUDA 双包；启动自检；可切换 device。
2. 大规模数据卡顿
   - 应对: 缩略图缓存、分页加载、异步 IO。
3. 模型版本污染
   - 应对: 版本快照不可变、产物目录隔离。
4. 标注误改
   - 应对: 编辑操作日志、撤销/重做（V0.2）。
