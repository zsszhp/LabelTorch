# 标炬（LabelTorch）PRD

## 1. 产品目标

为工业缺陷检测工程师提供一款本地离线桌面工具，显著降低 YOLO 数据准备、训练配置、版本迭代和辅助标注流程的复杂度，减少人工出错率。

## 2. 目标用户

1. 单用户工业算法工程师
2. 使用 Windows + NVIDIA GPU 的模型训练用户

## 3. 核心痛点

1. 标注工具与训练工具割裂，数据转换繁琐。
2. 多模型版本与权重管理混乱。
3. 增量迭代中辅助标注效率低。
4. 环境部署复杂，容易在依赖层失败。

## 4. 产品范围

### In Scope (MVP)

1. YOLO txt 数据导入与校验
2. 类别映射与 class 顺序重排
3. bbox 标注审阅与手工编辑（改框、删框、改类、补框）
4. 训练配置（v5/v8/v8_obb/v10/v11）
5. 早停（默认 patience=50，可调）
6. 模型版本管理与增量训练（选择历史权重）
7. 辅助标注（候选框、阈值筛选、批量确认）
8. 模型导出（pt/onnx，参数可调）
9. Windows 绿色包（CPU + CUDA 双包）

### Out of Scope (MVP)

1. 多用户权限系统
2. 云端协作
3. COCO/VOC 等多格式互转
4. 多卡训练、超参搜索、实验对比、断点续训
5. 分类与异常检测任务

## 5. 功能需求

### F1 项目与数据管理

1. 新建本地项目
2. 导入图片与标签目录
3. 校验样本完整性与坐标合法性
4. 生成导入报告

### F2 类别体系与映射

1. 从标签自动抽取类别集合
2. 允许用户调整类别顺序
3. 允许 class id 重映射并批量写回

### F3 标注审阅与编辑

1. 图片浏览、缩放、拖拽
2. 矩形框新增/删除/移动/缩放
3. 类别切换
4. 批量保存

### F4 训练任务

1. 选择模型族与权重
2. 配置参数并启动训练
3. 实时日志与进度
4. 早停配置

### F5 模型版本与增量训练

1. 训练完成自动入库版本
2. 版本列表与指标展示
3. 选择历史版本权重继续训练

### F6 辅助标注

1. 历史模型推理候选框
2. 阈值筛选
3. 批量确认并落盘
4. 人工修正闭环

### F7 导出

1. 导出 pt
2. 导出 onnx（默认 `opset=13,dynamic=True,simplify=True`）
3. 导出日志与错误提示

## 6. 非功能需求

1. 离线可用，不依赖网络。
2. 100-10000 样本规模下稳定运行。
3. 训练异常不导致主界面崩溃。
4. 操作可恢复，关键步骤有确认与提示。

## 7. 成功指标

1. 单次训练配置时间降低 50% 以上。
2. 增量标注人效提升 30% 以上。
3. 典型流程（导入-训练-导出）一次通过率 > 95%。
