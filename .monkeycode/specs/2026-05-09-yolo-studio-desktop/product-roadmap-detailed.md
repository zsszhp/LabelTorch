# 标炬（LabelTorch）详细版本路线图

## 版本目标总览

1. V0.1：最小可用闭环（MVP）
2. V0.2：效率增强与可恢复性
3. V0.3：规模化训练能力
4. V0.4：任务类型扩展
5. V0.5：工程化与企业级稳定性
6. V1.0：成熟产品发布

## V0.1 MVP（4-6 周）

目标：完成“导入-标注修正-训练-辅助标注-增量训练-导出”闭环。

范围：

1. 数据导入：YOLO txt
2. 类别管理：标签抽取 + class 重映射
3. 标注编辑：bbox 增删改、类别切换
4. 训练：v5/v8/v8_obb/v10/v11，早停默认 50 可调
5. 版本：训练结果版本化
6. 辅助标注：阈值筛选+批量确认
7. 导出：pt/onnx
8. 发布：Windows CPU/CUDA 绿色双包

出门标准：

1. 首次使用无需安装 Python
2. 1000 张样本流程稳定执行
3. 导入到导出成功率 >= 95%

## V0.2（3-5 周）

目标：提升训练效率与异常恢复能力。

范围：

1. 断点续训
2. 自动混合精度 AMP
3. 实验对比页（指标、参数、时长）
4. 导入 COCO/VOC（单向导入）

出门标准：

1. 中断恢复可用
2. 实验对比可追溯

## V0.3（4-6 周）

目标：支持更大规模、更复杂训练。

范围：

1. 多卡训练
2. 超参搜索
3. OBB 交互增强
4. 数据健康度分析（类别分布、框尺寸分布）

出门标准：

1. 多卡训练稳定
2. 超参搜索结果可回放

## V0.4（4-6 周）

目标：扩展到多任务标注训练平台。

范围：

1. 分类任务
2. 异常检测任务
3. 多任务共用项目空间

## V0.5（3-5 周）

目标：工程治理与长期维护增强。

范围：

1. 插件化模型适配器
2. 配置模板市场（本地）
3. 异常诊断与一键导出诊断包

## V1.0（成熟版，4-8 周）

目标：稳定、可维护、可扩展的工业级桌面产品。

范围：

1. 完整回归测试矩阵
2. 跨版本兼容策略
3. 发布节奏与 LTS 维护策略
4. 关键路径性能优化完成

成熟标准：

1. 10000 张样本长期操作稳定
2. 关键崩溃率低于约定阈值
3. 升级不破坏历史项目数据
