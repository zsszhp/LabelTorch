# 标炬（LabelTorch）测试与发布治理

## 测试分层

1. 单元测试：数据解析、标签写回、配置校验
2. 集成测试：导入-训练-导出闭环
3. 端到端测试：辅助标注与增量训练
4. 稳定性测试：长时间训练与大规模浏览

## 发布门禁

1. 必须通过核心回归用例。
2. CPU 包与 CUDA 包都要可启动。
3. 导出 pt/onnx 均要验证可用。

## 缺陷分级

1. P0：崩溃、数据损坏、无法训练
2. P1：核心流程失败但可绕过
3. P2：功能异常但不阻断主流程
4. P3：体验与展示问题

## 版本节奏

1. 双周迭代
2. 每迭代末冻结 2 天回归
3. 重大版本设置 RC 阶段
