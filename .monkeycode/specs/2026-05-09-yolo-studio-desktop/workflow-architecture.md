# 标炬（LabelTorch）工作流架构

## 主流程 1：数据导入

1. 选择 images 与 labels 路径
2. 扫描并构建样本索引
3. 执行合法性校验
4. 生成导入报告
5. 进入标注审阅页

## 主流程 2：类别映射

1. 从标签抽取类别
2. 用户调整顺序与映射
3. 执行批量重写
4. 生成映射变更记录

## 主流程 3：训练

1. 选择模型族
2. 选择或不选择 parent version 权重
3. 填写超参
4. 启动训练子进程
5. 回收日志与指标
6. 产出 model version

## 主流程 4：辅助标注与增量

1. 选择历史模型
2. 执行推理生成候选框
3. 阈值筛选
4. 批量确认 + 人工修正
5. 形成新数据版本
6. 发起增量训练

## 主流程 5：导出

1. 选择版本
2. 选择导出格式与参数
3. 执行导出
4. 导出校验
5. 归档导出记录
